sse-server

(sse-server)(sse-server {:keys [host port headers], :or {host "127.0.0.1", port 5558, headers {}}})

Starts a new SSE server for a core. Starts immediately.

Options: :host The address to listen on (default 127.0.0.1) Currently does nothing; this option depends on an incomplete feature in Aleph, the underlying networking library. Aleph will currently bind to all interfaces, regardless of this value. :port The port to listen on (default 5558) :headers Additional headers to send with the reply. By default Content-Type is set to text/event-stream and Cache-Control to no-cache. If you do not expose your client web application behind the same host, you will probably need to add an Access-Control-Allow-Origin header here
